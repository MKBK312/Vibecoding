# MindLink AI — 个人知识内化 RAG 系统设计文档

## 1. 项目概述

**项目名称:** MindLink AI
**项目类型:** 本地个人知识内化 RAG 系统
**核心定位:** 帮助学生快速吸收 PDF 论文、技术文档、Word 和 Obsidian 笔记的 Web 应用

**核心功能:**
- 多源数据解析：深度扫描 PDF 论文与 Obsidian 笔记
- 引用溯源检索：AI 回答时精准定位原文出处，标注文件名与页码
- 结构化摘要生成：从海量资料自动生成个人知识大纲
- 流式对话交互：流畅的 AI 问答体验

**技术选型:**
- 后端：Python FastAPI + PostgreSQL + pgvector
- 前端：React
- Embedding：本地模型 (bge-small-zh, 384 维)
- 生成：本地 Ollama（优先）+ 云端 API（备选 Claude/OpenAI）
- 文件解析：PyMuPDF (PDF) + python-docx (Word) + 通用 Markdown 解析

**部署模式:** 本地优先 + 可选云端
**用户规模:** 个人使用
**知识库规模:** 小型 (< 500 篇文档)

## 3. 数据模型

### 3.1 documents 表 (文档元数据)

| 字段 | 类型 | 说明 |
|------|------|------|
| id | UUID | 主键 |
| title | VARCHAR(500) | 文档标题 |
| file_type | ENUM | pdf / docx / md / obsidian |
| file_path | VARCHAR(1000) | 原始文件路径 |
| file_hash | VARCHAR(64) | 文件 MD5，用于去重 |
| metadata | JSONB | 文档级元数据 (作者、日期、页数等) |
| summary | TEXT | AI 生成的结构化摘要 |
| created_at | TIMESTAMP | 创建时间 |
| updated_at | TIMESTAMP | 更新时间 |

### 3.2 chunks 表 (文本块)

| 字段 | 类型 | 说明 |
|------|------|------|
| id | UUID | 主键 |
| document_id | UUID | 外键 → documents.id |
| content | TEXT | 原始文本内容 |
| chunk_index | INT | 块在文档中的顺序 |
| page_number | INT | PDF 页码 (其他类型为 NULL) |
| created_at | TIMESTAMP | 创建时间 |

索引: (document_id, chunk_index)

### 3.3 embeddings 表 (向量)

| 字段 | 类型 | 说明 |
|------|------|------|
| id | UUID | 主键 |
| chunk_id | UUID | 外键 → chunks.id (唯一) |
| embedding | VECTOR(384) | bge-small-zh 为 384 维 |
| created_at | TIMESTAMP | 创建时间 |

索引: IVFFLAT (embedding)

## 4. 文件解析流程

```
文件上传
    │
    ▼
┌─────────────────────┐
│  文件类型检测        │
│  (.pdf/.docx/.md)  │
└─────────────────────┘
    │
    ▼
┌─────────────────────┐
│  文本提取            │
│  - PDF: PyMuPDF     │
│  - DOCX: python-docx│
│  - MD: 通用解析      │
└─────────────────────┘
    │
    ▼
┌─────────────────────┐
│  块分割策略          │
│  - PDF: 按页/段落    │
│  - DOCX: 段落        │
│  - MD: Markdown 块  │
│  - 块大小: 500-1000字│
└─────────────────────┘
    │
    ▼
┌─────────────────────┐
│  元数据提取          │
│  - PDF: 标题/作者/页数│
│  - DOCX: 标题等      │
│  - MD: YAML front-  │
│    matter           │
└─────────────────────┘
    │
    ▼
存储到 documents 表
    │
    ▼
并行调用 Embedding 模型
(bge-small-zh)
    │
    ▼
存储到 chunks + embeddings 表
```

## 5. 检索与生成流程

```
用户提问
    │
    ▼
┌─────────────────────┐
│  问题Embedding       │
│  使用相同 bge-small-zh│
└─────────────────────┘
    │
    ▼
┌─────────────────────┐
│  混合检索策略         │
│  - Top-K 语义检索    │
│  - 关键词 BM25 检索   │
│  - RRF 融合排序      │
│  (默认 Top-K = 5)   │
└─────────────────────┘
    │
    ▼
┌─────────────────────┐
│  获取相关块           │
│  - content          │
│  - 文档标题         │
│  - 页码/文件名      │
└─────────────────────┘
    │
    ▼
┌─────────────────────┐
│  构建 Prompt         │
│  System: 你是学术助手 │
│  User: 问题 + 上下文  │
└─────────────────────┘
    │
    ▼
┌─────────────────────┐
│  流式生成响应         │
│  - 本地 Ollama (优先) │
│  - 云端 API (备选)   │
│  - 支持 SSE 流式输出 │
└─────────────────────┘
    │
    ▼
返回: AI回答 + 参考来源列表
      (文件名 + 页码)
```

## 6. API 路由设计

```
FastAPI 路由
│
├── /api/v1/
│   ├── documents/
│   │   ├── POST   /upload          -- 上传文件，返回 document_id
│   │   ├── GET    /                -- 列出所有文档
│   │   ├── GET    /{id}            -- 获取文档详情（含摘要）
│   │   ├── DELETE /{id}            -- 删除文档
│   │   └── POST   /{id}/reindex    -- 重新索引文档
│   │
│   ├── search/
│   │   └── POST   /                -- 混合检索，返回相关块
│   │
│   ├── chat/
│   │   └── POST   /stream          -- 流式对话 (SSE)
│   │
│   └── config/
│       ├── GET    /models          -- 获取可用模型列表
│       └── PUT    /retrieval       -- 更新检索参数 (Top-K 等)
```

## 8. 技术依赖

### 后端 (Python)

- fastapi >= 0.100
- uvicorn
- sqlalchemy >= 2.0
- asyncpg (PostgreSQL 异步驱动)
- pgvector (向量扩展)
- pymupdf (PDF 解析)
- python-docx (Word 解析)
- sentence-transformers (Embedding)
- httpx (HTTP 客户端)
- sse-starlette (SSE 流式支持)

### 前端 (React)

- next.js >= 14
- @tanstack/react-query
- tailwindcss
- lucide-react (图标)
- react-dropzone (文件上传)

### 基础设施

- PostgreSQL 15+ (with pgvector extension)
- Ollama (本地模型运行)
