# MindLink AI 对话记录

**日期**: 2026-05-08
**项目**: MindLink AI — 本地个人知识内化 RAG 系统

---

## 一、项目初始需求

> MindLink AI 项目初始化提示词
>
> 角色定位：你是一位精通全栈开发的资深 AI 工程师，擅长构建高性能、高颜值的 RAG（检索增强生成）应用。
>
> 项目目标：从零构建名为 "MindLink AI" 的本地个人知识内化系统。该系统允许学生上传 PDF、技术文档、Word 和 Obsidian 笔记，通过本地 LLM 进行深度解析，并提供带引用溯源的流式对话体验。
>
> 技术栈要求：
> - 前端：Next.js 14+ (App Router), TypeScript, Tailwind CSS, Shadcn UI
> - 后端：FastAPI (Python 3.10+), LlamaIndex (核心 RAG 框架), ChromaDB (本地向量库)
> - AI 引擎：通过 Ollama 调用本地 Llama 3 或 DeepSeek 模型
>
> 核心功能实现细节：
> - 分栏布局：左侧控制中心（文件拖拽上传区、本地知识库状态、模型参数调节器），右侧主交互区（Markdown 渲染的流式对话框 + 可折叠"参考来源"组件）
> - 引用溯源：后端检索时提取元数据（文件名、页码、文本片段），前端点击来源卡片高亮显示
> - 视觉风格：极简学术风。Lucide React 图标，冷灰色和深蓝色为主
>
> 特别指令：确保代码模块化，添加中文注释。PDF 解析优先考虑准确获取页码信息。使用 /superpowers，brainstorming 技能；先生成项目文档，不编码。

---

## 二、技术决策

| 决策项 | 用户选择 |
|--------|----------|
| 架构路线 | **FastAPI + Next.js 14 双端**（非 Next.js 全栈） |
| 前端策略 | **全部重写**（不复用旧组件） |
| LLM 模型 | **DeepSeek-R1**（Ollama） |
| Embedding 模型 | **bge-large-zh-v1.5**（Ollama） |
| RAG 策略 | **B：手动解析 + LlamaIndex 查询引擎**（精细控制 PDF 页码） |
| 执行模式 | **Subagent-Driven**：分派 2 个子 Agent 并行构建前后端 + 1 个监督 Agent 交叉审查 |

---

## 三、MVP 优先策略

> "请你先做 MVP 版本，测试功能，复杂功能后续优化"

- 后端：完整 6 个文件（config/models/pipeline/engine/main.py + requirements.txt）
- 前端：单文件 page.tsx 包含全部 UI + SSE 逻辑（无 react-query、无 shadcn/ui、无独立组件文件）

---

## 四、MVP 构建与审查过程

1. 清理旧代码，创建 backend/ 和 frontend/ 目录结构
2. 后端 Agent：构建 FastAPI + LlamaIndex + ChromaDB 管线（6 文件，~600 行）
3. 前端 Agent：构建 Next.js 14 单页应用（10 文件，595 行 page.tsx）
4. 监督 Agent：交叉审查 API 契约一致性（零关键不匹配）
5. 修复审查发现的问题：消除重复单例、删除死代码、补全 DeleteResponse 类型
